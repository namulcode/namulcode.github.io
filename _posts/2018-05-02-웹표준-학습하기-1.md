# 웹표준 학습하기

## 웹표준

- 기본 = 표준 
- 시맨틱 웹 
- 데이타 기반, 테스트 주도 
- ponyfill : 하위호환성, 파편화 
- 이슈/버그 대응 
- 마음가짐 (방향성, 철학) 

### 용어 

[HTML Living Standard](https://html.spec.whatwg.org/multipage/)

## HTML5

- [유효성 검사](https://html5.validator.nu/)
- form markup
- link과 button, 그리고 event

CSS

- layout
- RWD
- CSS 분리와 구조 
- class와 data-*

Javascript (ECMAScript) 

- JIT
- {key:value}
- var foo = new bar(); 
- datatype 
- if, for, switch
- 할당과 참조 
- this
- scope
- function
- return 

DOM 

- HTML == Object
- set, get
- events : click, change, submit, scroll, load, etc 
- logic 처리 

App

- 모바일 웹, 하이브리드 앱, 네이티브 앱 


심화학습 

- 경험 (지금 당장)
- 선 표준, 후 대응
- ajax
- node.js + express.js
- node modules (ex, 전처리기)
- template engine
- 작게 만들어 활용성을 높히고, 적게 만들어서 기본에 집중할 수 있도록 (컴포넌트, 함수 분리, 테스트)
- 유명 라이브러리의 API 따라 만들기 
- 서비스되고 있는 화면의 스타일, 기능을 따라 만들기 
- 검증 (hint, lint)
- 표준 확인 (문서 읽기)
- 경험이 제일 중요 (개발자든 고객이든)
- micro UI (MSA와 비슷한 개념. logic-less)
- 역할의 변화 : 컨텐츠 모델링, 데이타 모델링, 마크업 디자이너 
- 겸손한 코드 : 적절한 낮춤, 점진적 향상 
- 끈끈한 테스트 
- 플랫폼, 해상도, 기능으로 분리 
- 느슨한 결합, 재사용
